超级可爱的回溯算法

回溯模版

public class Backtrack {

    public void backtracking (/*参数*/) {
        /*
          定义两个成员变量
          一个收集满足要求的 结果
          一个收集 作为中途尝试的值，进行回退
        */
        /*
        if (终止条件) {
            存放结果
            终止本层循环
            return;
        }
        遍历本层集合中的元素
        for (选择：本层集合中元素[树中节点孩子的数量就是集合的大小]) {
            处理节点;
            backtracking(路径，选择列表); // 递归
            回溯，撤销处理结果
        }
        */
    }
}

例题

做题之前，先学习一下多叉树的遍历，然后这里用一个{ 1 , 2 , 3 }的全排列作为Demo

多叉树遍历模版

/* 基本的 N 叉树节点 */
class TreeNode {
    int val;
    TreeNode[] children;
}

void traverse(TreeNode root) {
    for (TreeNode child : root.children)
        traverse(child);
}

Demo

public class Permution1 {
    public List<List<Integer>> permute(int[] nums){
        List<List<Integer>> res  = new ArrayList<>();
        List<Integer> path = new ArrayList<>();
        dfs(nums,-1,path,res);
        return res;
    }

    private void dfs(int[] nums, int index, List<Integer> path, List<List<Integer>> res) {
        if(path.size() == nums.length){
            return;
        }
        if(index != -1){
            path.add(nums[index]);
        }
        if(path.size() == nums.length){
           res.add(new ArrayList<>(path));
        }

        for (int i = 0; i < nums.length; i++) {
            dfs(nums,i,path,res);

        }
//        dfs(nums,0,path,res);
//        dfs(nums,1,path,res);
//        dfs(nums,2,path,res);


        if (index != -1) {
            path.remove(path.size() - 1);
        }

    }

    public static void main(String[] args) {
        List<List<Integer>> permute = new Permution1().permute(new int[]{1, 2, 3});
        permute.forEach(System.out::println);
    }
}

接下来考虑一种情况，将Demo中冗余代码合并，细节像这样子的

if (path.size() == nums.length) {
            res.add(new ArrayList<>(path));
            return;
        }

这样会发生什么？

会增加冗余，例如有：三个 1，1，1 ……

它会去继续判断新的分支，在return的时候会重复添加 path 到 res 中去

一开始我是很不理解的，为什么条件是 path.size() == nums.length 这里的 size 明明是4 为什么还能进行下去？

到这里之前不应该是 已经返回了吗？为什么还会走下去？

带着疑问，我继续研究了一下哈

原来，在进入第四层的时候，首先进入if判断，在进行 path 的添加，这时候 path.size =4

但是在 if 判断的时候， 就会终止条件了，因为它的 path.size = 3 了 ，这时候就会 return

它根本不能在进行添加。

bug 就在 if 判断里 ，判断的时候，会 add path 到 res 中，会多执行 3 次，所以冗余了 三个 1， 1， 1

改进

更改后的dfs

private void dfs(int[] nums, int index, List<Integer> path, List<List<Integer>> res) {
    if (path.size() == nums.length) {
        res.add(new ArrayList<>(path));
        return;
    }

    for (int i = 0; i < nums.length; i++) {
        path.add(nums[i]);
        dfs(nums,i,path,res);
        path.remove(path.size() - 1);
    }
}

这就是回溯的本质，回溯回溯，就是在当前条件下，

新增条件，添加到结果集中，去除条件，

新增另一个条件，添加到结果集中，

…

以此类推

最后，从结果集中选取合适的结果，返回。

进行剪枝

private void dfs(int[] nums,  List<Integer> path, List<List<Integer>> res) {
    if (path.size() == nums.length) {
        res.add(new ArrayList<>(path));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if(path.contains(nums[i])){
            continue;
        }
        path.add(nums[i]);
        dfs(nums,path,res);
        path.remove(path.size() - 1);
    }
}

题解代码

public class Permution2 {
    public List<List<Integer>> permute(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        List<Integer> path = new ArrayList<>();
        dfs(nums,  path, res);
        return res;
    }

    private void dfs(int[] nums,  List<Integer> path, List<List<Integer>> res) {
        if (path.size() == nums.length) {
            res.add(new ArrayList<>(path));
            return;
        }
        for (int i = 0; i < nums.length; i++) {
            if(path.contains(nums[i])){
                continue;
            }
            path.add(nums[i]);
            dfs(nums,path,res);
            path.remove(path.size() - 1);
        }
    }

    public static void main(String[] args) {
        List<List<Integer>> permute = new Permution2().permute(new int[]{1, 2, 3});
        permute.forEach(System.out::println);
    }
}
